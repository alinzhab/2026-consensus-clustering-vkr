# QD-selection: теория, эксперимент, интерпретация

Документ объясняет логику модификации субдискретизации ансамбля (QD-selection),
описывает экспериментальный дизайн и формулирует корректные выводы.

## Зачем нужен интеллектуальный выбор базовых кластеризаций

В задаче консенсус кластеризации дан пул из `M` базовых кластеризаций.
На каждый прогон выбирается подмножество `m < M`. Стандартный подход —
случайный выбор (`selection_strategy="random"`).

Случайный выбор имеет три структурных недостатка:

**1. Дубли.** Пул может содержать базовые кластеризации, почти идентичные друг другу
(например, два запуска k-means с разными seed, но одинаковой сходимостью). При
случайном выборе оба могут попасть в подансамбль, удвоив их вклад в co-association
матрицу. Это смещает матрицу в сторону мажоритарной структуры.

**2. Нестабильные разбиения.** Пул может содержать «плохие» разбиения — случайные
или нестабильные (k-means с расходящейся сходимостью). Случайный выбор включает их
с той же вероятностью, что и хорошие, добавляя шум.

**3. Игнорирование разнообразия.** Информативный ансамбль должен содержать
разнообразные разбиения, покрывающие разные аспекты структуры данных. Случайный
выбор не гарантирует это свойство, особенно при малом `m`.

## Как работает QD-selection

**Quality-Diversity (QD) selection** — жадный детерминированный алгоритм:

1. Вычислить матрицу попарного согласия между всеми `M` базовыми кластеризациями
   (Rand-like метрика, O(M²·n)).
2. Качество кластеризации `i` = среднее согласие с остальными M−1 кластеризациями.
   Высокое качество → кластеризация согласована с большинством ансамбля.
3. Начать с кластеризации максимального качества.
4. На каждом шаге добавить кластеризацию `j`, максимизирующую:
   ```
   score(j) = α · quality(j)  +  (1−α) · diversity(j | selected)
   ```
   где `diversity(j | selected)` = среднее (1 − agreement) с уже выбранными.
5. Повторять до `|selected| == m`.

Параметр `qd_alpha ∈ [0, 1]` управляет балансом:
- `qd_alpha = 1.0` → только качество (выбрать самые надёжные, возможно похожие);
- `qd_alpha = 0.0` → только разнообразие (максимально отличающиеся, возможно слабые);
- `qd_alpha = 0.5` → баланс (рекомендуется по умолчанию).

## Почему это важно для консенсус кластеризации

Co-association матрица `CA[i,j]` = доля базовых кластеризаций, объединяющих объекты
`i` и `j` в один кластер. Качество `CA` прямо зависит от качества подансамбля.
Дублирующие кластеризации смещают `CA` к мажоритарной структуре; нестабильные —
добавляют шум во все ячейки. QD-selection убирает оба эффекта одновременно.

Важно: QD-selection ортогональна механизму консенсуса. Она применима ко всем
четырём алгоритмам (HC, HC weighted, SDGCA, SDGCA modified) как независимый шаг
до построения co-association матрицы. Это позволяет проверить гипотезу:
«улучшение качества подансамбля улучшает консенсус независимо от выбора алгоритма».

## Что означает std=0 при QD-selection

При `selection_strategy="qd"` все `cnt_times` прогонов используют одно и то же
подмножество кластеризаций → SDGCA / HC детерминированы → std метрик ≈ 0.

Это не баг, а свойство: QD-selection устраняет вариабельность субдискретизации.
Сравнение std(random) vs std(qd) само по себе информативно:
- Большое std(random) → случайный выбор нестабилен, ансамбль неоднороден.
- Малое std(random) → пул однороден, QD даёт меньшее преимущество.

Для статистически значимого сравнения нужно несколько независимых seed
(а не cnt_times повторений одного seed в QD-режиме).

## Экспериментальный протокол

Скрипт: `experiments/run_qd_selection_analysis.py`

```bash
# Smoke-test (2 датасета, 1 прогон, ~1-5 минут)
python experiments/run_qd_selection_analysis.py --smoke

# Основной эксперимент
python experiments/run_qd_selection_analysis.py --m 20 --runs 10 --methods average complete

# Полный эксперимент (все датасеты, все linkage)
python experiments/run_qd_selection_analysis.py --m 20 --runs 20
```

Результаты сохраняются в:
- `results/qd_selection_analysis.tsv` — полная таблица (дозаписывается).
- `results/qd_selection_summary.md` — автоматический отчёт с таблицами.

## Анализ по типам данных

Колонка `dataset_type` в TSV позволяет разбить результаты по доменам:

| Тип | Описание | Гипотеза об эффекте QD |
|---|---|---|
| `real_bio` | Реальные биологические (Ecoli, GLIOMA, Lung) | Умеренный прирост: пулы часто однородны |
| `real_text` | Реальные текстовые (BBC) | Высокий прирост: текстовые ансамбли часто содержат похожие разбиения |
| `compact` | Синт. компактные, хорошо разделённые | Небольшой прирост: случайный выбор и так стабилен |
| `elongated` | Синт. вытянутые кластеры | Умеренный прирост: нестабильные разбиения по вытянутым кластерам |
| `mixed_complex` | Синт. сложная геометрия | Высокий прирост: ансамбль неоднороден, QD помогает |
| `high_dimensional` | Высокоразмерные данные | Риск переоценки quality: согласие в высокой размерности нестабильно |

Тип определяется автоматически из словаря `DATASET_TYPES` в `ensemble_selection.py`.
Для новых датасетов добавить запись в словарь вручную.

## Корректные выводы

**МОЖНО утверждать после эксперимента:**

- «QD-selection улучшило среднее NMI на X датасетах из Y при `qd_alpha=0.5` и `m=20`»
- «На датасетах типа `mixed_complex` и `elongated` прирост ΔNMI > 0 стабильно»
- «std метрик при QD-selection в N раз меньше, чем при random: случайный выбор
  нестабилен на данном ансамбле»
- «Эффект QD-selection статистически значим по критерию Вилкоксона на парах
  (algo_random vs algo_qd) при p < 0.05»

**НЕЛЬЗЯ утверждать без дополнительных тестов:**

- «QD-selection универсально лучше random» — нет теоретической гарантии при
  малом пуле (M = m: QD вырождается в полный выбор).
- «Прирост объясняется балансом quality/diversity» — нужен аблационный анализ:
  `qd_alpha=1.0` (только quality) vs `qd_alpha=0.5` vs `qd_alpha=0.0` (только diversity).
- «QD-selection заменяет подбор гиперпараметров» — нет, `qd_alpha` сам является
  гиперпараметром и может требовать подбора.
- «Результаты обобщаются на произвольные ансамбли» — выводы применимы к пулам
  базовых кластеризаций, сгенерированных способом, аналогичным использованному в
  экспериментах.

## Связь с задачей ВКР

Тема: «Разработка системы тестирования иерархических алгоритмов консенсус
кластеризации». QD-selection усиливает работу по нескольким осям:

1. **Инженерный вклад**: расширение системы тестирования новым режимом
   субдискретизации — параметр `selection_strategy` в каждом алгоритме.
2. **Исследовательский вклад**: первое систематическое сравнение random vs
   QD-selection на 4 алгоритмах, 4 linkage, ~12 датасетах в единой системе.
3. **Практический вклад**: пользователь системы может выбрать стратегию субдискретизации
   в зависимости от размера и однородности пула базовых кластеризаций.
