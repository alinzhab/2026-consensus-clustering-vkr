# Ablation study для SDGCA modified в системе тестирования консенсус кластеризации

## Зачем нужна ablation study

В дипломной работе заявляется **система тестирования иерархических алгоритмов консенсус кластеризации**. Сравнение «SDGCA vs SDGCA modified» в одну строку таблицы **недостаточно**: в модифицированной версии одновременно меняются несколько механизмов (веса кластеров, способ построения cannot-link, опционально масштаб диффузии, опционально внешний отбор ансамбля). Без декомпозиции невозможно ответить на вопросы комиссии: *что именно дало эффект? был ли он за счёт честного алгоритма или за счёт подбора гиперпараметров / отбора базовых разбиений?*

Ablation study разделяет:

1. **Внутренние модификации** SDGCA modified (fuzzy-веса, heat-kernel, adaptive diffusion time) при **фиксированном протоколе** выбора столбцов `members`.
2. **Внешний протокол** QD-selection — отбор подмножества из `m` базовых кластеризаций из пула `M`. Это **не** часть SDGCA modified; его эффект измеряется отдельными контрастами (D−A, E−B, F−C).

## Какие компоненты SDGCA modified проверяются (по коду)

| Компонент | Базовый SDGCA | SDGCA modified | Заменяемый шаг |
|-----------|---------------|----------------|----------------|
| Веса кластеров в NWCA | `compute_neci` — энтропия по жёстким меткам в ансамбле | `compute_modified_neci` — нечёткая энтропия по `X` (если есть), структурный множитель `1/log₂ K_i`, нормировка по размеру кластера | Построение вектора весов перед `compute_nwca` |
| Cannot-link / диссимилярность кластеров | `random_walk_of_cluster` (итеративное RW) | `graph_diffusion_of_cluster` — `exp(−t L_norm)` (heat-kernel) | Матрица `compute_d` → `compute_d_diffusion` |
| Масштаб диффузии `t` | Не применимо (RW с фиксированным числом шагов) | Фиксированный `t` (например 1.0) или **adaptive diffusion time**: эвристика `t = clip(1/λ₂, [0.1, 5])` по спектральному зазору нормированного лапласиана графа сходства кластеров | Только в ветке с ADMM (`η ≤ 1`); при `η > 1` диффузия не вызывается |

**Важно:** adaptive diffusion time — это **эвристика**, не оптимальный в смысле минимизации потерь; в тексте ВКР и в коде используйте формулировку *adaptive diffusion time*, не «оптимальное t».

## Почему нельзя ограничиться сравнением «SDGCA vs SDGCA modified»

1. **Per-dataset `DEFAULT_PARAMS`** в `sdgca_modified.py` даёт разным датасетам разные `(λ, η, θ, t)` — это смешивает эффект алгоритма с эффектом подстройки и создаёт риск data leakage в протоколе. В ablation-скрипте основной режим — **generic defaults** (те же базовые значения, что у SDGCA: `λ=0.09, η=0.75, θ=0.65`, для modified дополнительно `t=1.0` в варианте fixed). Варианты G/H с `tuned` помечены отдельно и **не** должутся основным доказательством превосходства.

2. **QD-selection** меняет входной ансамбль до входа в SDGCA. Утверждать «модификация SDGCA помогла», смешивая её с QD, **некорректно** без контраста «тот же отбор + тот же SDGCA».

## Экспериментальный дизайн (варианты A–F)

Фиксируются: один и тот же файл датасета, одно `linkage`, один `seed`, одно `m`, одно число прогонов `runs`, один `qd_alpha` (для QD). Случайный отбор столбцов при одинаковом `seed` даёт **одинаковое** подмножество столбцов на каждом прогоне между A/B/C (см. реализацию `run_sdgca` / `run_sdgca_modified`).

| Вариант | Алгоритм | Параметры | Отбор ансамбля |
|---------|----------|-----------|----------------|
| A | SDGCA baseline | generic | random |
| B | SDGCA modified | generic, fixed `t=1.0` | random |
| C | SDGCA modified | generic, adaptive `t` | random |
| D | SDGCA baseline | generic | QD |
| E | SDGCA modified | generic, fixed `t` | QD |
| F | SDGCA modified | generic, adaptive `t` | QD |

Интерпретация контрастов:

- **B − A** — эффект внутренних модификаций (при random), без QD и без adaptive t.
- **C − B** — эффект **только** adaptive diffusion time при прочих равных.
- **D − A** — эффект QD на baseline.
- **E − B** — эффект QD на modified fixed.
- **F − C** — эффект QD на modified adaptive.

## Метрики и артефакты

Скрипт `experiments/run_sdgca_modified_ablation.py` сохраняет:

- `results/sdgca_modified_ablation_<timestamp>.tsv` — полный прогон;
- копии: `results/sdgca_modified_ablation_latest.tsv`, `results/sdgca_modified_ablation.tsv`;
- `results/sdgca_modified_ablation_<timestamp>.md` и `results/sdgca_modified_ablation_summary.md`.

В TSV, помимо NMI/ARI/F-score и runtime: `memory_peak_mb` (пик Python heap по `tracemalloc` на один вызов), `n_clusters_ensemble` (сумма числа кластеров по выбранным `m` столбцам), `diffusion_time_used` для adaptive.

## Как интерпретировать результаты

**Корректно:**

- Указывать долю датасетов, где ΔNMI превышает малый порог (например 0.002), **в разрезе типа данных** (`dataset_type` из `ensemble_selection.py`).
- Отдельно обсуждать runtime и память; если adaptive t почти не увеличивает время при том же n — это сильный инженерный аргумент.
- Ссылаться на подгонку `time ≈ a · n^b` в summary как **эмпирическую** оценку масштаба, не как теорему.

**Некорректно:**

- «Модификация улучшает SDGCA» без указания варианта (B vs A? C vs A? с QD или без?) и без таблицы/графика.
- Перенос выводов с `tuned` (G/H) на честное сравнение A–F.
- Смешивание эффекта QD с эффектом heat-kernel / fuzzy-весов в одном предложении без ссылок на контрасты D−A, E−B, F−C.

## Визуализация

Страница `/analytics` загружает `sdgca_modified_ablation_latest.tsv` и строит графики: среднее NMI по вариантам, ΔNMI (C−B), runtime, heatmap «тип данных × вариант», эффект QD (E−B, F−C), распределение adaptive t, **runtime vs n**.

## Дальнейшие идеи модификаций SDGCA (не реализованы здесь)

- Адаптивный порог `τ` в cannot-link вместо фиксированного 0.8.
- Замена `expm` на полиномиальную аппроксимацию или чебышёвский ряд для ускорения на больших K.
- Явная калибровка `t` по валидации на hold-out подвыборке объектов (дорого, но честнее, чем эвристика по λ₂).
