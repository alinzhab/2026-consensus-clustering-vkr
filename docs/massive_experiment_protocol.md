# Massive Synthetic Experiment Protocol

Тема ВКР: **Разработка системы тестирования иерархических алгоритмов консенсус кластеризации**.

Цель протокола - честно проверить, когда консенсус кластеризация полезна, дает ли она выигрыш относительно одиночных алгоритмов кластеризации, и как качество связано с типом данных, временем работы и вычислительной сложностью.

## Зачем 500 датасетов

Один или несколько датасетов легко случайно "любят" конкретный алгоритм. Большая синтетическая коллекция нужна, чтобы покрыть разные типы структуры:

- compact: простые хорошо разделимые кластеры;
- overlapping: перекрывающиеся кластеры;
- imbalanced: разные размеры кластеров;
- high_dimensional: высокая размерность;
- elongated: вытянутые и несферические кластеры;
- density_varied: неоднородная плотность;
- mixed_complex: стресс-тест с несколькими сложностями одновременно.

## Почему dataset-level cross-validation

Обычная supervised cross-validation делит объекты внутри одного датасета. Для этой ВКР это не главный протокол, потому что алгоритмы кластеризации работают без обучения на метках объектов. Если делить объекты, мы проверяем не обобщение алгоритма на новые типы данных, а устойчивость внутри уже выбранной геометрии.

Здесь делится множество датасетов:

- train/design: проектирование и отладка протокола;
- validation: выбор одного глобального профиля параметров;
- test: финальная проверка без изменения параметров;
- fold_id: 5-fold dataset-level CV для дополнительных оценок устойчивости.

Ground truth используется только для финального расчета NMI, ARI и F-score. Он не используется для построения разбиений и не используется для подбора параметров на test split.

## Как предотвращается подгонка

Нельзя выбирать параметры отдельно под каждый датасет. Вместо этого заранее задаются глобальные кандидаты:

- m: 8, 12, 20;
- selection_strategy: random, qd;
- qd_alpha: 0.3, 0.5, 0.7;
- linkage: average, complete;
- SDGCA modified: fixed diffusion_time=1.0 или adaptive diffusion time как отдельный режим.

На validation split выбирается один глобальный профиль. На test split этот профиль применяется без изменений.

## Что можно утверждать

Можно утверждать только то, что подтверждено на test split:

- где консенсус кластеризация лучше одиночных алгоритмов;
- где она хуже;
- для каких типов данных QD-selection помогает;
- оправдан ли рост runtime;
- какие типы данных являются failure cases.

## Что нельзя утверждать

Нельзя утверждать, что алгоритм "лучше вообще", если:

- выигрыш найден только на validation;
- параметры подбирались по test;
- использовались per-dataset tuned параметры как основной результат;
- улучшение есть только на одном типе данных;
- статистические тесты не подтверждают отличие.

Если метрики плохие, это не ошибка протокола. Это результат, который нужно объяснить через overlap, high dimensionality, imbalance, density variation, качество базового ансамбля или неподходящее linkage.
