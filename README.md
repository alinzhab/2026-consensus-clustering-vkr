# 2026_Consensus_Clustering

Репозиторий содержит программную реализацию системы тестирования алгоритмов консенсус кластеризации на реальных и синтетических датасетах. В проекте реализованы базовая и модифицированная версии иерархической консенсус кластеризации, алгоритм `SDGCA` и его модифицированный вариант, генераторы синтетических данных, а также веб-интерфейс для запуска вычислительных экспериментов, расчёта метрик качества и сравнительного анализа результатов.

## Постановка задачи

Дано множество базовых кластеризаций `M = {C_1, ..., C_m}` одного набора объектов. Требуется построить итоговое разбиение `C*`, агрегирующее информацию ансамбля, и оценить качество по метрикам `NMI`, `ARI`, `pairwise F-score` относительно эталонной разметки. Цель работы — разработать и сравнить варианты консенсус-кластеризации, предложить две модификации (взвешенная иерархическая, модифицированный SDGCA с нечёткой энтропией и графовой диффузией) и провести их статистически обоснованное эмпирическое сравнение.

## Быстрый старт

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# Один эксперимент
python consensus_lab\sdgca.py --dataset Ecoli --m 40 --runs 20 --method average

# Полный пакетный анализ
python consensus_lab\run_full_analysis_suite.py --runs 3 --m 20

# Статистическая значимость различий между алгоритмами
python consensus_lab\statistical_tests.py --metric nmi_mean --baseline sdgca_modified

# Тесты
pip install pytest
pytest tests\

# Веб-интерфейс
python app.py
```

### Почему каталог называется `consensus_lab`

Прежнее название `python_port` появилось исторически — на старте проекта планировался порт некоторых алгоритмов с MATLAB. Фактически `consensus_lab/` содержит самостоятельные реализации на Python с собственными модификациями и единым интерфейсом, поэтому имя `python_port` вводило в заблуждение. Текущее имя `consensus_lab` отражает назначение каталога — исследовательская «лаборатория» алгоритмов консенсус-кластеризации.

## Реализованные алгоритмы

### 1. Базовая иерархическая консенсус кластеризация

Алгоритм строит стандартную матрицу совместной встречаемости объектов по ансамблю базовых кластеризаций, после чего применяет иерархическую агрегацию для получения итогового разбиения

### 2. Модифицированная иерархическая консенсус кластеризация

В данной версии используется взвешивание базовых разбиений. На основе этих весов формируется взвешенная consensus-матрица, которая затем используется на этапе финальной иерархической агрегации

### 3. SDGCA

`SDGCA` (`Similarity and Dissimilarity Guided Co-association Matrix Construction`) — алгоритм консенсус кластеризации, который использует информацию как о сходстве, так и о различии между объектами и кластерами при построении итоговой consensus-матрицы

### 4. Модифицированный SDGCA

В модифицированной версии реализованы следующие изменения:

- нечёткое энтропийное взвешивание кластеров
- графовая диффузия вместо схемы случайных блужданий
- поддержка финальной иерархической агрегации с правилами `single`, `complete`, `average` и `ward`

## Поддерживаемые правила иерархической агрегации

Во всех основных алгоритмах поддерживаются следующие правила межкластерного объединения:

- `single`
- `complete`
- `average`
- `ward`

## Формат датасетов

### Формат `.mat`

Минимально ожидаемые поля:

- `members` — матрица базовых кластеризаций
- `gt` — истинные метки классов

Дополнительно может присутствовать:

- `X` — матрица признаков объектов

### Формат `.npz`

Ожидаемые поля:

- `X` — матрица признаков объектов
- `gt` — истинные метки классов
- `members` — ансамбль базовых кластеризаций
- `meta` — параметры генерации

## Синтетические генераторы

### `densired_style_generator.py`

Генератор синтетических данных со сложной кластерной структурой. Поддерживаются следующие свойства:

- неоднородная плотность
- шум
- вытянутые кластеры
- ветвления
- нетривиальная геометрия кластеров

### `repliclust_style_generator.py`

Генератор архетипных синтетических данных с интерпретируемыми параметрами:

- число кластеров
- размерность пространства
- степень перекрытия
- различие форм и масштабов
- дисбаланс размеров кластеров

## Веб-интерфейс

Веб-приложение включает следующие основные страницы:

- главная страница
- страница загрузки датасетов
- страница генерации синтетических данных
- страница тестирования алгоритмов
- страница просмотра результатов

Через веб-интерфейс можно:

- загрузить собственный датасет
- сгенерировать синтетический датасет
- сохранить наборы данных в каталог `datasets/`
- выбрать алгоритм и параметры запуска
- запустить вычислительный эксперимент
- сохранить результаты в каталог `results/`

## Метрики качества

Во всех основных экспериментах рассчитываются следующие метрики:

- `NMI`
- `ARI`
- `F-score`

Для каждой метрики сохраняются:

- среднее значение
- стандартное отклонение по нескольким запускам
