# Consensus Clustering Test System

Веб-система для тестирования алгоритмов консенсусной кластеризации. Проект позволяет загружать пользовательские датасеты, генерировать синтетические выборки, строить ансамбль базовых разбиений, запускать несколько алгоритмов в одинаковых условиях и сохранять метрики экспериментов.

Система разрабатывалась в рамках ВКР по теме тестирования иерархических алгоритмов консенсусной кластеризации.

## Что реализовано

- базовая иерархическая консенсусная кластеризация по матрице совместной встречаемости;
- взвешенная иерархическая версия с оценкой надёжности базовых разбиений;
- SDGCA (Similarity and Dissimilarity Guided Co-association Matrix Construction);
- модифицированный SDGCA с нечётким энтропийным взвешиванием и графовой диффузией;
- QD-отбор подансамбля по критериям качества и разнообразия;
- построение базовых разбиений по матрице признаков;
- генераторы синтетических данных с разной геометрией кластеров;
- веб-интерфейс на Flask;
- агент рекомендаций, который анализирует датасет и предлагает стартовые параметры запуска.

## Структура проекта

```text
.
├── app.py                         # Flask-приложение

├── datasets/                      # датасеты

```

Основная логика находится в каталоге `consensus_lab/`:

- `hierarchical_consensus.py` — базовая иерархическая консенсусная кластеризация;
- `hierarchical_consensus_modified.py` — взвешенная иерархическая версия;
- `sdgca.py` — реализация SDGCA;
- `sdgca_modified.py` — модификация SDGCA с нечётким взвешиванием и графовой диффузией;
- `base_clusterings.py` — построение ансамбля базовых разбиений;
- `ensemble_selection.py` — QD-отбор подансамбля;
- `metrics.py` — NMI, ARI и pairwise F-score;
- `dataset_diagnostics.py`, `smart_recommender.py` — диагностика датасета и рекомендации параметров;
- `simple_dataset_generator.py`, `qiu_joe_style_generator.py`, `repliclust_style_generator.py`, `densired_style_generator.py` — генераторы синтетических данных.

## Быстрый запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python app.py
```

После запуска веб-интерфейс будет доступен локально:

```text
http://127.0.0.1:5000
```

## Запуск алгоритмов из командной строки

Примеры одиночных запусков:

```powershell
python consensus_lab\hierarchical_consensus.py --dataset Ecoli --m 20 --runs 20 --method average
python consensus_lab\hierarchical_consensus_modified.py --dataset Ecoli --m 20 --runs 20 --method average --sharpen 1.5
python consensus_lab\sdgca.py --dataset Ecoli --m 20 --runs 20 --method average
python consensus_lab\sdgca_modified.py --dataset Ecoli --m 20 --runs 20 --method average
```

Поддерживаемые правила иерархической агрегации:

- `single`;
- `complete`;
- `average`;
- `ward`.

## Форматы данных

Система работает с наборами данных в форматах `.mat`, `.npz`, `.csv`, `.tsv`, `.txt`, `.json`.

Для запуска консенсусных алгоритмов нужен ансамбль базовых разбиений:

- `members` — матрица размера `n x M`, где строки соответствуют объектам, а столбцы — базовым разбиениям;
- `gt` — эталонные метки, если они есть;
- `X` — матрица признаков, если нужно построить ансамбль автоматически или использовать модификацию SDGCA;
- `meta` — служебная информация о наборе данных.

Если в загруженном файле нет `members`, но есть матрица признаков `X`, система может построить ансамбль базовых разбиений автоматически.

## Синтетические данные

В проекте реализованы четыре генератора:

- простой гауссовский генератор для компактных кластеров;
- генератор на основе идей Qiu--Joe для управления разделимостью и ковариационной структурой;
- генератор на основе подхода Repliclust для высокоуровневой настройки формы, размерности, перекрытия и дисбаланса;
- генератор на основе идей DENSIRED для сложной плотностной структуры, вытянутых форм и ветвлений.

Синтетические данные используются для проверки алгоритмов в контролируемых условиях: можно отдельно исследовать перекрытие, дисбаланс, высокую размерность, вытянутую форму и неоднородную плотность.

## Веб-интерфейс

Веб-приложение включает страницы для основных этапов работы:

- просмотр и загрузка датасетов;
- генерация синтетических выборок;
- запуск алгоритмов;
- просмотр сохранённых результатов;
- аналитика и сравнение запусков;
- рекомендации параметров через агент.

Агент рекомендаций анализирует характеристики датасета: размер, размерность, число классов, дисбаланс, внутренние индексы качества, плотность и другие диагностические признаки. На основе этих данных он предлагает стартовые параметры: размер подансамбля, стратегию построения базовых разбиений, правило агрегации и параметры для SDGCA.

## Метрики качества

При наличии эталонной разметки система рассчитывает:

- NMI (Normalized Mutual Information);
- ARI (Adjusted Rand Index);
- pairwise F-score.

Для серий запусков сохраняются средние значения и стандартные отклонения метрик.

## Хранение данных

Локальный режим использует SQLite и файловую систему:

- `datasets` — сведения о зарегистрированных наборах данных;
- `results` — параметры запусков и полученные метрики;
- файлы датасетов хранятся отдельно в каталоге `datasets/`;
- результаты экспериментов сохраняются в каталоге `results/`.

## Тесты

```powershell
pytest tests
```

Smoke-проверки и отдельные сценарии запуска находятся в корне проекта и каталоге `experiments/`.

## Переменные окружения

Для базового локального запуска переменные окружения не требуются. Если используется агент с внешней LLM, ключи доступа задаются через `.env` или переменные окружения. Файл `.env` не должен попадать в репозиторий.
