# Развёртывание на Render

Проект подготовлен для бесплатного деплоя на Render.

## Что уже настроено

- `requirements.txt` — зависимости Python
- `render.yaml` — конфигурация сервиса Render
- `app.py` — запуск через переменные окружения `PORT`, `FLASK_HOST`, `FLASK_DEBUG`

## Как выложить сайт

1. Загрузить проект в GitHub.
2. Зайти в [Render](https://render.com/).
3. Создать `New +` -> `Blueprint`.
4. Подключить GitHub-репозиторий с этим проектом.
5. Render сам прочитает `render.yaml`.
6. Подтвердить создание сервиса.
7. Дождаться сборки и получить публичную ссылку вида `https://...onrender.com`.

## Что важно знать

- Бесплатный тариф Render подходит для демонстрации, но не для серьёзной постоянной нагрузки.
- Сервис может “засыпать” после простоя и просыпаться с задержкой.
- Файлы, загруженные через веб-интерфейс, и локально сохранённые результаты не являются надёжным постоянным хранилищем.
- Если нужен постоянный storage, позже лучше вынести данные в внешнее хранилище или базу данных.

## Локальная проверка перед деплоем

```bash
pip install -r requirements.txt
python app.py
```

или production-старт:

```bash
gunicorn app:app --bind 0.0.0.0:5000
```
